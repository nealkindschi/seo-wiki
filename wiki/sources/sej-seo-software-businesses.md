Viola Eva, "Effective SEO Implementation for Software Businesses" (Search Engine Journal, 2019-10-29).

## Key takeaways

- Extends [[saas-seo-strategy]] with a **site-architecture/navigation**
  layer not yet covered by that playbook's funnel/page-type/content-
  format sections — this source is about how the whole site's
  information architecture should be organized, not which individual
  pages to build.
- **Mega menu strategy**: present the same product through multiple
  navigation lenses (brand/feature name, functionality/keyword,
  industry vertical, user role, specific task) to match different
  customer avatars' mental models — but **link every overlapping
  category to the same underlying page** rather than creating a
  separate page per menu path. This is a concrete UI-level mechanism
  for avoiding the exact keyword-cannibalization problem
  [[keyword-mapping-and-cannibalization]] already diagnoses — the fix
  here happens at the navigation-design stage, before the
  cannibalization exists.
- **Siloing structure with a concrete depth/volume model**: Level 1
  (core/product pages) → Level 2 (silos: tutorials, resources, reviews,
  content categories) → Level 3 (individual posts, **10-30 minimum per
  silo**) — a more specific version of
  [[link-and-anchor-text-best-practices]]'s existing "siloing +
  topic clusters, combined" guidance, which didn't specify a
  page-count threshold for a silo to gain topical relevance.
- **Most search traffic comes from bottom-level (Level 3) posts**
  optimized for specific keywords, not the top-level product/category
  pages — consistent with, and reinforces,
  [[gofishdigital-seo-for-saas]]'s "single-vendor product pages don't
  rank for competitive terms" finding: both sources point traffic
  expectations toward long-tail/informational content rather than
  top-of-hierarchy pages.
- Older source (2019), no statistics/case studies cited — treat as a
  structural framework contribution, not empirical evidence.

Samuel Schmitt, "Topic Clusters Case Study: Key Insights" (samuelschmitt.com, published 2020-11-03, updated 2023-02-28).

## Key takeaways

- Case study: a 3,500-word Google Tag Manager tutorial was split into a
  five-page topic cluster (one pillar + subpages, hierarchical URLs,
  bidirectional internal linking, 301 redirect from the original URL to
  the pillar). Weekly pageviews went from ~50-80 to ~2,000 (1000%
  growth), 16,000 pageviews over five months, with traffic sustained and
  still growing at time of writing.
- Diagnostic method: Google Search Console keyword-to-section mapping
  showed content performance degrading deeper into the original
  long-form article — the first section ranked for 59% of top-10
  keywords, later sections far fewer. This ranking-degradation pattern
  is the signal used to decide *when* to split a page into a cluster.
- One subpage ("Goals and Events"), which had zero top-10 rankings as
  part of the original monolithic article, became a top cluster
  performer after being split out — format change alone, not new
  content, drove the visibility gain.
- 7-step implementation process: analyze structure/intent → map
  keywords to sections via GSC → keyword-gap research → rewrite
  titles/headings from keyword data → build hierarchical URLs
  (`/pillar-topic/subtopic/`) → bidirectional pillar↔subpage internal
  linking → publish + 301-redirect the original URL to the pillar.
- Caveat: best suited to tutorial-style content with distinct,
  separable search intents per section — not a universal recommendation
  to split all long-form content. Long-form and clustered content each
  have valid uses.
